#### 一、内容概述与要求

​	参加C语言程序设计课程考试的考生应理解C程序的基本构成、C程序中的数据类型及其运算、结构化程序设计的三种基本结构（顺序、选择、循环）、数组、函数；了解编译预处理命令、指针、结构体与共用体、位运算等知识模块所涉及的基本概念、语法规范；掌握运用以上语法知识进行结构化程序设计的基本方法；熟悉程序设计中简单的数据结构和算法，具有一定的分析和编写简单C程序的能力，以及基本的纠错和调试程序的能力。考生应具有规范的结构化编程思维，能够综合运用所学知识分析并解决简单的实际问题。

#### 二、知识要点与考核要求

- ##### C语言程序的结构

  - ###### 知识范围

    1. ###### C语言程序的基本构成；

       简单的C程序一般具有如下形式：

       ```c
       #include <stdio.h>							//指令
       int main(int argc, char const* argv[]){		//函数
           return 0;								//语句
       }
       ```

       C程序依赖3个关键的语言特性：

       1. ###### 指令(在编译前修改程序的编辑命令)

          在编译C程序之前，**预处理器**会首先对其进行编辑，将预处理器执行的命令称为**指令**

          - 预处理指令

            大多数预处理指令都属于下面三种类型

            - ###### 宏定义	

              #define指令定义一个宏，#undef指令删除一个宏定义

            - ###### 文件包含

              #include指令导致一个指定文件的内容被包含到程序中

            - ###### 条件编译

              #if、#ifdef、#ifndef、#elif、#else和#endif指令可以根据预处理器测试的条件来确定是将一段文本块包含到程序中还是将其排除在程序之外

            - ###### 指令都以**#**开始

            - ###### 在指令的符号之间可以插入任意数量的空格或水平制表符

              ```c
              # define 			N 				100	//合法
              ```

            - ###### 指令总是在第一个换行符处结束，除非明确的指明要延续

              ```c
              #define DISK_CAPACITY (SIDE *				\
              						TRACKS_PER_SIDE *	\
              						SECTORS_PER_TRACK *	\
              						BYTES_PER_SECTORS)
              ```

              如果想在下一行延续指令，我们必须在当前行的末尾使用 \ 字符

            - ###### 指令可以出现在程序中的任何地方

              通常将#include和#define放在开头，其他则放在后面

            - ###### 注释与指令可以放在同一行

       2. ###### 函数(被命名的可执行代码块，如main函数)

       3. ###### 语句(程序运行时执行的命令)

    2. ###### 头文件、函数构成以及程序中的注释；

       - ###### 头文件

         定义：如果想让几个源文件访问相同信息，可以把此信息放入一个文件中，然后利用#include指令把该文件的内容带进每个源文件中，把按照此种方式包含的文件叫做头文件。

         头文件的扩展名为.h

         ###### #include指令

         #include指令，此指令使得在任意数量的源文件中共享信息成为可能，这些信息可以是函数原型、宏定义、类型定义等

         #include指令告诉预处理器打开指定文件，并将指定文件内容插入到当前文件中。

         书写格式：

         【#include <文件名>】	用于C语言自身库的头文件

         【#include  "文件名"】	用于其他所有头文件，自身编写的头文件

       - ###### 函数构成

       - ###### 注释

    3. ###### C程序的执行过程；

    4. ###### 源程序的书写格式。
  - 考核要求
    1. ###### 理解C程序的基本结构及执行过程。
    2. ###### 理解C程序中头文件、注释等概念，会在程序中正确使用文件包含命令以及添加注释。
    3. ###### 理解C函数的构成形式。
